=======================================================
Container Documentation
=======================================================

This container uses OpenRC instead of systemd and includes
mise for managing programming language runtimes.

AVAILABLE DOCUMENTATION
=======================

1. Service Management (OpenRC)
   See: /docs/services.txt
   
   Learn how to:
   - Create and manage OpenRC services
   - Set up service dependencies
   - Configure service runlevels
   - Handle logging and process supervision

2. Language Runtimes (mise)
   See: /docs/language-runtimes.txt
   
   Learn how to:
   - Use pre-installed languages (Node.js, Python, Ruby, etc.)
   - Install additional language versions
   - Set project-specific versions
   - Manage multiple languages in one project

QUICK TIPS
==========

- If you tried to use systemctl/systemd, read /docs/services.txt
- To check installed languages: mise list
- To create a service: create script in /etc/init.d/ and add to runlevel
- To use a specific Node version: mise use node@20
- Check running services: rc-status
- Start a service: rc-service servicename start

For detailed information, please refer to the documentation files above.
