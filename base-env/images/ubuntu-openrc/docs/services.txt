=======================================================
OpenRC Service Management - Quick Start
=======================================================

OpenRC is already installed and configured in this container.

CREATING A BASIC SERVICE
========================

1. Create a service script:
   nano /etc/init.d/myservice

2. Add this basic template:
   #!/sbin/openrc-run
   
   name="myservice"
   description="My service description"
   command="/path/to/your/program"
   command_args="--your-options"
   pidfile="/run/${RC_SVCNAME}.pid"
   
   depend() {
       need net
   }

3. Make it executable:
   chmod +x /etc/init.d/myservice

4. Add to default runlevel:
   rc-update add myservice default

5. Start the service:
   rc-service myservice start

BASIC COMMANDS
==============

Service Control:
rc-service myservice start     # Start service
rc-service myservice stop      # Stop service  
rc-service myservice restart   # Restart service
rc-service myservice status    # Check status

System Status:
rc-status                      # Show running services
rc-update show                 # Show all services and runlevels

Add/Remove Services:
rc-update add myservice default    # Add to default runlevel
rc-update del myservice default    # Remove from runlevel

SIMPLE EXAMPLE
==============

Creating a web server service:

/etc/init.d/webserver:
#!/sbin/openrc-run

name="webserver"
description="Simple Python web server"
command="/usr/bin/python3"
command_args="-m http.server 8080"
command_background="yes"
pidfile="/run/${RC_SVCNAME}.pid"
directory="/var/www"

depend() {
    need net
}

Then run:
chmod +x /etc/init.d/webserver
rc-update add webserver default
rc-service webserver start

That's it! Your service will start automatically when the container starts.
